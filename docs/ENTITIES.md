# Entities

This document describes the backend domain entities and their persistence shape. The application supports two persistence backends through repository interfaces:

- SQLite/TypeORM entities in `api/src/**/*.entity.ts`
- MongoDB/Mongoose schemas in `api/src/**/*.schema.ts`

`DatabaseModule.forRoot()` selects MongoDB when `DATABASE=mongodb`; otherwise it uses SQLite with `SQLITE_PATH` or `database.sqlite`.

## Persistence Notes

- Repository interfaces are the application contract. Controllers and services should depend on repository tokens, not directly on TypeORM or Mongoose models.
- TypeORM entities use UUID primary keys named `id`.
- Repository records often expose Mongo-style `_id` for service compatibility, even when backed by SQLite.
- Some SQLite fields store structured data as JSON strings because SQLite does not have the same native document shape as MongoDB.
- Mongoose schemas generally use `{ timestamps: true }`, which creates `createdAt` and `updatedAt` even when those properties are not declared directly on the class.
- Secret values are intentionally separated from secret metadata at the API/service boundary.

## User

Represents an application user account.

Source files:

- `api/src/users/user.entity.ts`
- `api/src/users/user.schema.ts`
- `api/src/users/user.repository.ts`

Fields:

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` / `_id` | string | yes | UUID in SQLite; Mongo document id in MongoDB. |
| `username` | string | yes | Unique. Mongo stores it lowercased and trimmed. |
| `email` | string | yes | Unique. Mongo stores it lowercased and trimmed. |
| `password` | string | yes | Password hash. Do not expose this field in API responses. |
| `role` | string | yes | Defaults to `user`. |
| `createdAt` | Date | generated | Generated by TypeORM or Mongoose timestamps. |
| `updatedAt` | Date | generated | Generated by TypeORM or Mongoose timestamps. |

Main usage:

- Authentication and JWT subject lookup.
- User administration.
- Role assignment by role name.

## Password Reset

Represents a one-time password reset token.

Source files:

- `api/src/auth/password-reset.entity.ts`
- `api/src/auth/password-reset.schema.ts`
- `api/src/auth/password-reset.repository.ts`

Fields:

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` / `_id` | string | yes | UUID in SQLite; Mongo document id in MongoDB. |
| `userId` | string | yes | References the user requesting the reset. |
| `token` | string | yes | Unique reset token. |
| `expiresAt` | Date | yes | Expiration time. MongoDB has a TTL index on this field. |
| `used` | boolean | yes | Defaults to `false`; set to `true` after reset. |
| `createdAt` | Date | generated | Generated by TypeORM or Mongoose timestamps. |

Main usage:

- `forgotPassword` creates a new token and deletes previous tokens for the same user.
- `resetPassword` validates token existence, expiration, and usage before changing the password.

Represents an imported API project and the MCP server configuration generated from it.

Source files:

- `api/src/swagger/swagger-project.entity.ts`
- `api/src/swagger/swagger-project.schema.ts`
- `api/src/swagger/swagger-project.repository.ts`
- `api/src/dynamic-mcp/types.ts`

Fields:

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` / `_id` | string | yes | UUID in SQLite; Mongo document id in MongoDB. |
| `name` | string | yes | Project/server display name. |
| `baseUrl` | string | yes | Base URL for generated HTTP tools. |
| `description` | string | no | Optional project description. |
| `version` | string | no | API/project version. |
| `shareSlug` | string | no | Unique human-readable slug used in generated public share URLs. It is generated automatically from the server name and can be edited while remaining unique. |
| `rawSpec` | object/string | no | Original OpenAPI/Swagger spec. SQLite stores it as text; Mongo stores it as an object. |
| `tools` | `GeneratedTool[]` | yes | Generated MCP tools. SQLite stores JSON text; Mongo stores objects. |
| `auth` | `AuthConfig` | yes | Upstream API auth configuration. Defaults to `{ "type": "none" }`. |
| `status` | string | yes | Defaults to `active`; Mongo enum allows `active`, `inactive`, and `error`. |
| `errorMessage` | string | no | Last import/runtime error message. |
| `mcpApiKey` | string | no | Legacy single MCP API key. |
| `mcpApiKeys` | `McpApiKeyEntry[]` | yes | Multi-key MCP access list. |
| `resources` | `McpResource[]` | yes | MCP resources exposed by this server. |
| `prompts` | `McpPrompt[]` | yes | Server-specific MCP prompts. |
| `chains` | `ToolChain[]` | yes | Multi-step tool workflows. |
| `oauthClientId` | string | no | OAuth client id for OAuth flows. |
| `oauthClientSecret` | string | no | OAuth client secret. Treat as sensitive. |
| `tags` | string[] | yes | Project labels. SQLite stores JSON text. Source type is stored as a `source:<type>` tag, for example `source:rest`. |
| `rateLimit` | object | yes | Defaults to disabled with `requestsPerMinute: 60`. |
| `isPaused` | boolean | yes | When true, MCP endpoints should return unavailable behavior. |
| `maintenanceMode` | object | yes | Optional maintenance response configuration. |
| `availabilityWindow` | object | yes | Time-window access control. |
| `alertConfig` | object | yes | Error threshold and notification config. |
| `tenantConfig` | object | yes | Multi-tenant parameter injection config. |
| `connectionConfig` | `DbConnectionConfig` | no | Present in SQLite entity and repository contract. |
| `dbQueries` | `DbQuery[]` | no | Legacy storage for data-source operations. Each entry can include `parameters`, `inputSchema`, and `outputSchema` so it can be exposed as an MCP Tool or Resource. |
| `createdAt` | Date | generated | Generated by TypeORM or Mongoose timestamps. |
| `updatedAt` | Date | generated | Generated by TypeORM or Mongoose timestamps. |

Persistence note:

- `connectionConfig` and `dbQueries` are declared in both SQLite and MongoDB persistence models and must remain aligned because database-backed tools depend on them.
- Server templates are REST templates and must create servers with the `source:rest` tag.
- Data-source operations should carry `inputSchema` and `outputSchema`; generated MCP Tools should inherit those schemas when available.

Source files:

- `api/src/ai-providers/ai-provider.entity.ts`
- `api/src/ai-providers/ai-provider.schema.ts`
- `api/src/ai-providers/ai-provider.repository.ts`

Fields:

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` | string | yes | UUID in SQLite; Mongo document id is mapped by repository records. |
| `name` | string | yes | Human-readable provider configuration name. |
| `description` | string | no | Optional usage note. |
| `provider` | string | yes | Provider key such as `openai`, `anthropic`, `google`, `azure-openai`, `ollama`, or `custom`. |
| `model` | string | yes | Provider model/deployment name. |
| `apiKey` | string | yes | Write-only credential for API-backed providers. Ollama can store an empty string. Never expose in API responses. |
| `baseUrl` | string | no | Optional override endpoint. Required for Azure OpenAI deployment URLs; optional for default provider APIs. |
| `isActive` | boolean | yes | Active providers can be tested or used for generation. Defaults to `true`. |
| `isDefault` | boolean | yes | Default active provider used when an AI generation request does not pass `providerId`. Only one provider should be default. |
| `lastTestStatus` | `success` / `error` | no | Last connection-test outcome. |
| `lastTestedAt` | Date | no | Last connection-test timestamp. |
| `lastTestError` | string | no | Last connection-test error summary. |
| `createdAt` | Date | generated | Generated by TypeORM or Mongoose timestamps. |
| `updatedAt` | Date | generated | Generated by TypeORM or Mongoose timestamps. |

Security and usage notes:

- API responses omit `apiKey`; provider execution happens only server-side.
- Testing and AI-assisted generation require `ai_providers_execute`, separate from provider CRUD permissions because execution can consume external model credits.
- The REST server creation flow can send endpoint metadata to `/api/ai-providers/generate-tools`; it does not send upstream API credentials.

## Settings

Represents global application settings. This is a singleton record keyed by `global`.

Source files:

- `api/src/settings/settings.entity.ts`
- `api/src/settings/settings.schema.ts`
- `api/src/settings/settings.repository.ts`

Fields:

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` / `_id` | string | yes | UUID in SQLite; Mongo document id in MongoDB. |
| `key` | string | yes | Unique singleton key. Defaults to `global`. |
| `serverBaseUrl` | string | yes | Public server URL used in generated examples and password reset links. |
| `defaultTimeoutMs` | number | yes | Default timeout for tool HTTP calls. Defaults to `30000`. |
| `smtpHost` | string | yes | SMTP host for password reset email. |
| `smtpPort` | number | yes | Defaults to `587`. |
| `smtpUser` | string | yes | SMTP username. |
| `smtpPass` | string | yes | SMTP password. Treat as sensitive. |
| `smtpFrom` | string | yes | Sender address for email. |
| `jwtSecret` | string | yes | JWT signing secret persisted for runtime auth, OAuth/MCP bearer tokens, and share links. Treat as sensitive; safe settings responses only expose `jwtSecretSet`. Falls back to `JWT_SECRET` from the process environment when empty. |
| `globalRequestHeaders` | `{ name: string; value: string }[]` | no | Headers injected into outbound requests. SQLite may store `null`; Mongo defaults to `[]`. |
| `observabilityEnvironment` | `Record<string, string>` | no | Saved Observability page environment-control draft, including feature flags, log level, service identity, metrics path, and OTEL exporter settings. SQLite stores it as JSON; Mongo stores it as an object. |
| `termServer` | string | no | User-facing terminology override for the Server concept. |
| `termTool` | string | no | User-facing terminology override for the Tool concept. |
| `termResource` | string | no | User-facing terminology override for the Resource concept. |
| `termPrompt` | string | no | User-facing terminology override for the Prompt concept. |
| `termChain` | string | no | User-facing terminology override for the Chain concept. |
| `termSecret` | string | no | User-facing terminology override for the Secret concept. |

Terminology note:

- The frontend `TerminologyProvider` loads these optional fields from `/api/settings`.
- Empty or `null` terminology fields fall back to the active i18n locale defaults.
- When adding a configurable term, update `SettingsRecord`, both persistence models, both settings repositories, `src/context/TerminologyContext.tsx`, settings UI, locale defaults, and this document.

Observability environment note:

- The Observability runtime page loads and saves `observabilityEnvironment` through `/api/settings`.
- Saved values are a deployment/configuration draft for operators and do not mutate environment variables inside an already-running backend process.
- Saving this field reuses the existing `settings_manage` permission because it is persisted global settings data.

JWT secret note:

- The Settings page can save a replacement JWT signing secret in `jwtSecret`; blank form submissions keep the current saved value.
- Safe settings reads do not expose `jwtSecret`; they expose `jwtSecretSet`.
- If no `jwtSecret` is saved, auth falls back to the process `JWT_SECRET` value for bootstrap compatibility.
- Rotating the saved secret invalidates previously signed sessions, OAuth/MCP bearer tokens, and share links.

## Secret

Represents a named secret value used by projects/tools.

Source files:

- `api/src/secrets/secret.entity.ts`
- `api/src/secrets/secret.schema.ts`
- `api/src/secrets/secret.repository.ts`

Fields:

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` | string | yes | UUID in SQLite; Mongo document id is mapped by repository records. |
| `name` | string | yes | Unique secret name. |
| `value` | string | yes | Secret value. Treat as sensitive and avoid logging. |
| `description` | string | no | Optional human-readable description. |
| `createdAt` | Date | generated | Generated by TypeORM or Mongoose timestamps. |
| `updatedAt` | Date | generated | Generated by TypeORM or Mongoose timestamps. |

API/service boundary:

- List/read operations return `SecretMetadataRecord`, which omits `value`.
- `GET /api/secrets/:id/value` is the dedicated reveal endpoint and requires `secrets_reveal_values`.
- Create/update operations return metadata only.
- Internal server-side resolution for MCP execution can still access values through `ISecretRepository`.

## Maintenance Checklist

When changing entities:

1. Update the TypeORM entity, Mongoose schema, repository contract, and both repository implementations when applicable.
2. Keep SQLite JSON-string fields and Mongo object/array fields semantically aligned.
3. Update this document when fields are added, renamed, removed, or their meaning changes.
4. Review tests for both persistence modes when behavior is shared by repository interfaces.
